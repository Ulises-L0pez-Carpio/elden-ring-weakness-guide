# Case Study

## Overview

*Elden Ring Weakness Guide* started as a reference problem: valuable gameplay information existed, but it lived across spreadsheet tabs, mixed visual layouts, and fragmented notes that were not practical to consult during play.

The project goal was to turn that material into a mobile companion app that felt faster, clearer, and more intentional than the original sources.

## The Challenge

The source material created three product problems:

1. the information was dense and structurally inconsistent
2. some sections behaved like datasets, while others depended on visual formatting
3. the reading flow was too slow for moment-to-moment decision making

## My Approach

I treated the work as a combination of product design, interface design, and data translation.

- I reorganized the content around player tasks instead of source tabs.
- I reduced the visible information to the fields that actually help with decisions.
- I built a stronger visual system so the app felt like a finished product rather than a spreadsheet wrapper.

## The Solution

The final app was organized into focused mobile sections:

- Nightlords
- Night Bosses
- Recluse
- Scholar
- iFrames and recovery

That structure made it easier to move from search to inspection to comparison, with clearer cards, stronger section identity, and a more game-aligned presentation layer.

## Key Decisions

- Design for mobile-first consultation, not desktop-style browsing.
- Prioritize lookup speed over exhaustive raw-data display.
- Use section-specific visual language to improve recognition and navigation.
- Keep the app scalable so new data sources could be integrated without breaking the interface model.

## Outcome

The result was an Android-ready companion app that combined:

- curated boss weakness reference
- structured encounter browsing
- Nightlord detail views
- themed section layouts
- polished branded presentation

The current public release line presented in this project is `3.0.1 / 31`.
